# 受众矩阵 / Audience matrix

Read the row(s) for the persona(s) you are reporting to **before** writing the
page. Each row is the rubric for that page's first screen (SKILL.md §2, §4,
Quality bar #1). "营销人会遇到的任何角色" — extend this table when a new one
appears; keep the columns identical.

Columns: **关心什么** · **首屏先讲** · **必须砍掉/下沉** · **语气** · **TA 一定会问的那个问题** · **常见翻车**.

---

## 老板 / CEO · 创始人

- **关心什么**: 增长是否高效、趋势是否在变好、要不要追加投入、有没有风险。本质是资本配置问题。
- **首屏先讲**: 一句可被 TA 复述给董事会的结论 + 钱的方向（ROI / CAC / payback / pipeline 贡献）。
- **砍掉/下沉**: 渠道操作细节、工具与平台名词、创意过程、内部排期。
- **语气**: 直接、有判断、敢给建议。不要只汇报，要给一个「我建议 X」。
- **TA 会问**: "所以我们该多投还是收手？这个数健康吗？"
- **常见翻车**: 首屏堆渠道明细；只报喜不报趋势；没有一句话结论；没有 ask。

## 直属上级 / 市场 VP · 总监

- **关心什么**: 对既定目标的达成度、偏差原因、风险、需要 TA 帮忙解除的阻塞、要的资源。
- **首屏先讲**: 目标红黄绿仪表 + 最大缺口 + 需要的支持（预算/人/决策）。
- **砍掉/下沉**: 战术执行流水账、已知背景。
- **语气**: 务实、负责、把「我需要什么」说清楚，不甩锅也不报平安。
- **TA 会问**: "离目标差多少？为什么？你需要我做什么？"
- **常见翻车**: 把状态当成果；风险藏在最后；要资源含糊其辞。

## 产品 PM

- **关心什么**: 营销信号对产品/路线图的含义 —— 哪些需求被验证、转化在哪一步卡住、要不要共建。
- **首屏先讲**: 用户与需求信号、转化漏斗卡点、建议的产品×营销共建项。
- **砍掉/下沉**: 创意稿、媒介排期、品牌叙事。
- **语气**: 协作、以用户为中心、用证据不用感觉。
- **TA 会问**: "用户到底想要什么？产品要不要改什么来接住这波？"
- **常见翻车**: 通篇营销 KPI 没有产品含义；不给共建建议。

## 销售 / 客户成功 (CS)

- **关心什么**: 现在能不能用、给哪些客户、怎么跟进、线索质量。要可落地的弹药。
- **首屏先讲**: 可直接用的素材清单 + 线索量与质量 + 推荐话术/异议处理。
- **砍掉/下沉**: 品牌长期战略、营销内部反思。
- **语气**: 赋能、可立即落地、给「拿去就能用」的东西。
- **TA 会问**: "我今天能拿什么去打单？这些线索靠谱吗？"
- **常见翻车**: 给战略不给弹药；线索没有质量分层；没有话术。

## 媒体 / 记者 · PR

- **关心什么**: 有没有新鲜、可引用、可独立核实的事实。TA 在替读者把关。
- **首屏先讲**: 一句新闻点 + 可引用的数据/里程碑 + 事实出处与发言人。
- **砍掉/下沉**: 内部 KPI、未公开数字、营销黑话、自夸形容词。
- **语气**: 客观、可核实、零浮夸；提供可直接引用的事实块。
- **TA 会问**: "这有什么新闻价值？这个数字谁背书、能不能引用？"
- **常见翻车**: 营销稿冒充新闻稿；给不能公开的数;无出处；形容词轰炸。
- **发布**: 这类页面通常 `--public`（会被外部链接/转载）。

## 客户 / Customer

- **关心什么**: 这对「我」意味着什么、是否可信、风险多大、我接下来做什么。
- **首屏先讲**: 对客户的价值一句话 + 可由客户自行验证的证据 + 明确的下一步。
- **砍掉/下沉**: 内部指标、内部路径、内部人名与工具。
- **语气**: 以客户为中心、降低风险、可审计；提供「你可以自己验证」的路径。
- **TA 会问**: "这对我有什么用？凭什么信你？我要做什么？"
- **常见翻车**: 通篇讲自己多牛；证据客户无法独立验证；泄露内部信息。
- **发布**: 常 `--public` 或定向链接；务必过 talk-html 可移植性检查。

## 投资人 / 董事会

- **关心什么**: 市场大小、护城河、可重复的增长引擎、单位经济、节奏。
- **首屏先讲**: 增长引擎与单位经济（CAC/LTV/payback）+ 可重复性证据 + 趋势。
- **砍掉/下沉**: 单次活动八卦、战术细节、内部流程。
- **语气**: 战略、数字诚实（坏消息也讲）、有节奏感、像季度叙事。
- **TA 会问**: "这是可重复的引擎还是一次性运气？单位经济成立吗？"
- **常见翻车**: 把单次成功当引擎；只挑好数；没有可重复性论证。

## 合作伙伴 / 渠道

- **关心什么**: 共赢有多大、各自要做什么、节奏与依赖。
- **首屏先讲**: 共同收益 + 双方各自下一步动作 + 时间节奏。
- **砍掉/下沉**: 内部预算、内部反思、与本伙伴无关的战略。
- **语气**: 互惠、清晰分工、对等。
- **TA 会问**: "我能得到什么？我要做什么？什么时候？"
- **常见翻车**: 只讲自己收益；分工含糊；把伙伴当执行方而非对等方。

---

## 选择与组合规则

- **单受众**: 选一行，整页只为 TA 写。首屏 = 该行「首屏先讲」。
- **多受众**: 仅当 ≥2 个真实读者共用同一批证据时，用 audience tab/锚点，每个
  tab 是该受众的 1–5 段，证据（block 3）共享，「诚实边界/数据来源」共享。
  禁止为显得周全而虚构第二受众（talk-html §3.2 + SKILL.md Quality bar #5）。
- **拿不准受众**: 先问用户「报给谁」。无法得到答案时默认「直属上级/VP」并在
  页面注明（SKILL.md Failure modes）。
- **新角色**: 营销人可能遇到法务、合规、HR、社区、KOL/达人、经销商……出现新
  角色时按同样六列补一行，规则不变：先讲 TA 关心的、砍掉 TA 不关心的、用 TA
  的语气、回答 TA 一定会问的那个问题。
